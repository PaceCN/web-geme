# 庄园物语安全整理

当前项目分两层：

- `src/`：Cloudflare Pages 静态前端原型
- `backend/`：Axum 后端和运营后台原型

## 现在已经处理

- 后台管理接口需要登录 token。
- 后台支持 `HUAYUAN_ADMIN_USER` / `HUAYUAN_ADMIN_PASSWORD` 环境变量覆盖默认账户。
- 广告恢复体力在后端有广告位、凭证、每日次数、体力上限校验。
- 小游戏开关由后台管理端控制。
- 后台操作写入审计日志。

## 仍然必须补强

- 默认账号 `admin / admin123!` 只能本地测试用，部署前必须改。
- 当前后台数据是内存存储，重启会丢失；正式版需要 SQLite / PostgreSQL。
- 当前密码哈希是 SHA-256 原型；正式版应换成 Argon2id / bcrypt。
- 当前 token 是服务端内存 session；正式版建议存数据库或 Redis，并增加退出登录接口。
- 管理端必须放在 HTTPS 后面，不要明文 HTTP 暴露到公网。
- `/api/admin/*` 必须限制 CORS 来源，只允许管理端域名。
- 广告奖励不能只相信前端按钮，必须接广告平台服务端回调或可验证 token。
- 小游戏分数不能只信前端，后端至少要做分数范围、局时长、频率、用户状态校验。
- 玩家金币、体力、背包、订单、任务进度都必须以后端为准。

## 广告奖励安全流程

```text
前端展示激励广告
广告 SDK 返回完成事件
前端拿到广告平台凭证
前端请求 POST /api/v1/ads/reward
后端校验广告位、凭证、次数、体力上限、账号状态
后端发放体力并写审计日志
```

## 小游戏结算安全原则

- 前端可以负责手感和动画。
- 后端必须负责最终奖励。
- 前端上传 `game_id`、`score`、`duration_ms`、`client_nonce`。
- 后端根据游戏开关、分数上限、提交频率、账号状态决定是否发奖。
- 检测异常时只拒绝奖励，不要让前端直接改背包或金币。
