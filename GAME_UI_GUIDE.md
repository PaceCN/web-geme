# 怎么让庄园物语看起来像一个游戏

## 核心方向

不要把页面做成“信息列表 + 按钮”。手机游戏的第一眼应该是：

- 顶部像 HUD：头像、体力、金币、经验都像游戏资源条。
- 中间像场景：玩家在庄园里操作，而不是在表格里点功能。
- 卡片像任务面板：订单、背包、商店都是游戏面板，不是普通网页列表。
- 小游戏是独立画面：打开后应有专属棋盘、分数、剩余步数、结算按钮。
- 所有功能入口要有图标或素材：少用纯文字按钮。

## 当前 UI 约束

- 主平台是手机竖屏。
- 主框架固定：顶部 HUD、内容场景、底部 5 栏导航。
- 小游戏统一从 `src/modules/` 接入。
- 收益只考虑广告，广告入口应该像“补给站”而不是充值页。
- 所有面板圆角保持 8px，避免廉价大圆角卡片堆叠。

## 建议素材

- 顶部庄园远景：作为 HUD 背景。
- 任务面板边框：木牌、羊皮纸、金属边框均可，但不要过黄。
- 道具图标：统一 256x256 透明 PNG/WebP。
- 小游戏棋子：开心消消乐至少 5 套颜色和形状，到十用木牌数字。
- 底部导航：每个 tab 使用明确图标，不要只放文字。

## 画面层次

```text
背景层：庄园/庭院/夜色氛围
HUD 层：头像、体力、金币、广告补给
任务层：订单、背包、商店、个人信息
小游戏层：全屏或弹层棋盘
反馈层：奖励、toast、动效、结算
```

## 后续改 UI 的优先级

1. 先替换真实素材：顶部背景、任务横幅、小游戏棋子。
2. 给三个小游戏加消除/合成动画。
3. 把订单和背包从文字列表改成图标网格。
4. 增加领取奖励弹窗，而不是只显示 toast。
5. 增加音效开关和震动反馈。

## 不建议做的事

- 不做营销式落地页。
- 不做大面积浅黄色网页卡片。
- 不做纯表格后台风格的玩家前端。
- 不把多个卡片嵌套在一起。
- 不用过多渐变球、装饰光斑或无意义插画。
